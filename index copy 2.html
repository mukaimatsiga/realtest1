HTML Journal Entry:

What does HTML Stand for?

  • HyperText Markup Language

    • What is HTML in my own words?

  • HTML is the "bones" of a website - it displays the text, pictures, links, etc.
  • You can find HTML code on websites by clicking the "view" --> "Developer" --> "view source"

    • CSS Journal Entry:

    What is CSS?

  • It stands for Cascading Style Sheets
  • CSS allows you to change the font color, size of the font, add animations, etc

    • Color Journal Entry:

    When should we use text for colors compared to HEX?

  • HEX helps us find more specific colors - text is quicker
  • HEX color values give us more options to be more specific when choosing a color for our site
  • I decided to download the colorpicker package on Atom which has made color selection a lot easier

    • HEX for black:

  • #000 or #000000

    • HEX for white:

  • #fff or #ffffff

    • It's easier to read HEX values in lower case rather than upper case, however it is easiest to copy and paste the values from a tool

    Font and Fontstack Journal Entry:

  • Google Fonts is available online for us to get other fonts that aren't generic and on our personal system
  • A font stack is part of a font family, it makes it so our font has a backup font for our website, just in case Google Fonts goes down
  • You can look up different fonts that work well together for choose if you have a good font style or not

    • Github Review Journal Entry:

    What is Git:
  • Git is a version control system that is used to track changes to files

    • What is GitHub:
  • Company that provides services for Git Repos

    • What are GitHub Pages:
  • The service that hosts your website for others to look at

    • What is GitHub Desktop:
  • The desktop version of GitHub

    • What is a Repository (or repo):
  • A folder that holds all of your HTML/CSS files

    • What is a Remote Repo:
  • Your repo stored in the cloud (ohhhh...the cloud....)

    • What is a commit:
  • A commit is when you save your code

    • What is a push:
  • A push is when you move your commit to the public?

    • What is a clone:
  • When you copy someone else's code for you to review and edit

    • What is a fork:
  • Forking is when you copy someone else's code when it is still connected to their repo for them to view and review

    • Managing GitHub Journal Entry:

    Why is it important to post commits often for your projects?
  • Pushing your commits helps you save your progress in the cloud, so your work is saved, even if you arent on your working computer

    • What is the purpose of a README.md file?
  • README.md file helps people understand the project - it's good for first time users

    • Why would you fork a project?
  • Forking allows you to connect your original project to edited work without disrupting the original

    • Why would you clone a project?
  • Cloning a project helps you use another developers code without disrupting or attaching anything to it

    • Why would you revert to an older version?
  • Reverting a document helps you work on a project and try things out without having to worry that you will permanently mess up the code
